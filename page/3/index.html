MYSQL常用基本命令结构化查询语言SQL（Structured Query Language）

发表于 2016-03-20 | 分类于 Linux | 阅读次数

SQL语句分类：

  • DDL（Data Definition Language）数据定义语言：CREATE ALTER DROP
  • DCL（Date Control Language）数据控制语言：GRANT REVOKE COMMIT ROLLBACK
  • DML（Date Manipulation Language）数据操作语言：SELECT INSERT DELETE UPDATE

mysql5.1及以前默认引擎为MyISAM ，mysql5.5.5以后默认为InnoDB

常用命令：

  • 启动mysql命令：/etc/init.d/mysqld start
  • 查看mysql端口： ss -lnt|grep 3306
  • 查看mysql进程：ps -ef|grep mysql|grep -v grep
  • 优雅关闭数据库的方法：mysqladmin -uroot -p123456 shutdown
  • 配置文件修改登录提示符，在my.cnf [mysql]模块下添加如下内容，无需重启，退出当前session重新登录即可：prompt=\u@主机名 \r:\m:\s->
  • 多实例登录：（1）本地：mysql -u -p -S /data/3306/mysql.sock （2）远程：mysql -u -p -h -P3306
  • 添加超级管理员：grant all privileges on . to system@’localhost’ identified by ‘123456’ with grant option;
  • 设置密码：mysqladmin -u root password ‘123456’;
  • 多实例修改密码：mysqladmin -uroot -p’123456’ password ‘ ‘ -S /data/3306/mysql.sock
    或者在MYSQL里面用sql语句：UPDATE mysql.user SET password=PASSWORD(“123456”) WHERE user=’root’ and host=’localhost’;
    修改单用户密码，set password=PASSWORD(“123456”);
  • 忘记root密码修改方法：停止mysql服务，使用–skip-grant-tables启动MYSQL，用上述后两种方法修改密码：mysqld_safe –skip-grant-tables –user=mysql &（多实例需要指定配置文件：mysqld_safe –defaults-file=/data/3306/my.cnf –skip-grant-tables & 需要注意参CREATE DATABASE 数据库名 CHARACTER SET GBK COLLATE gbk_chinese_ci;数前后顺序） 再用空密码登录mysql
  • 创建数据库：create database 数据库名; show databases like ‘ ‘;
  • 指定字符集：show character set;
    CREATE DATABASE 数据库名 CHARACTER SET GBK COLLATE gbk_chinese_ci;
  • use <数据库名> 相当于linux cd命令
    select user(); 相当于whoami select database(); 相当于pwd
  • drop user “user”@”主机域” 删除用户 drop database <数据库名>; 删除数据库
  • grant all on dbname.* to username@’localhost’ identified by ‘passwd’; flush privileges; show grants for ‘username’@’localhost’; IP授权形式： 10.0.0.% 10.0.0.0/255.255.255.0
  • 回收权限：REVOKE INSERT ON . FROM ‘jeffrey’@’localhost’; flush privileges; show grants for ‘username’@’localhost’;
    常规情况下授权 select insert delete update 4个权限即可。
  • desc <表名>; 查看表结构。 show create table <表名>; 查看创表的语句。
  • primary key(id) 主键 一般主键配自增属性 auto_increment KEY index_name(name) 普通索引
    增加普通索引：alter table <表名> add index 索引名(字段); 删除索引把add换成drop
    对字段前N个字符创建普通索引：create index 索引名 on 表名(列(N));
    联合索引具有前缀性，经常用于条件查询的放前面。
  • insert into <表名> [(<字段名1>,<字段名2>..)] values( , ); 数字可以不加引号，字符串需要加引号
  • select <字段1，字段2…> from <表> where 条件 （limit 0,2;查部分数据，从第0行开始查两行）（order by 排序 desc降序 asc升序）（INTO OUTFILE ‘/tmp/a.txt’导出数据到文件）
  • explain +select语句 获取select的相关信息，查看是否用索引，做优化：
    （1）抓慢查询：
    现场抓：mysql -uroot -p123456 -S /data/3306/mysql.sock -e “show full processlist;”|grep select -i
    分析慢查询日志：
    long_query_time = 1
    log-slow-queries = /data/3306/slow.log
    log_queries_not_using_indexes
    每天零点定时任务切割日志：
    cd /data/3306
    /bin/mv slow.log /backup/$(date +%F)_slow.log
    mysqladmin -uroot -p123456 -S /data/3306/mysql.sock flush-logs
    （2）explain语句检查索引执行情况
    （3）对需要建索引的列建立索引
    （4）分析慢查询的工具，每天发邮件。工具：mysqlsla
  • profile 查询优化 SQL语句执行细节
  • update 表名 set 字段=新值，…… where 条件（一定要注意加条件）
    登录mysql时加入 -U 参数，使用安全模式，防止误更新 alias mysql=’mysql -U’
  • delete from 表名 where 条件表达式
  • truncate table 表名 速度更快，直接清空对应数据的物理文件。
  • alter table 表名 add 字段 类型 其他;
  • rename table 原表名 to 新表名;
  • show variables like “character_set%”; 查看mysql字符集

  • mysqldump 备份数据库命令，恢复用mysql命令重定向<
    mysqldump -uroot -p123456 -S /data/3306/mysql.sock 库名 > 备份路径.sql
    备份完成检查：egrep -v “#|*|–|^$” 备份数据库文件路径
    （1）-B 参数：表示接多个库，会加入建库及use库的过程。没有特殊情况一般都要加此参数。加入-B参数，后面可以直接接多个库名进行备份
    （2）–compact 优化输出的大小，让容量更少，适合调试用。
    （3）mysqldump …|gzip > 路径.sql.gz 一般加入gzip管道压缩备份数据库
    （4）–default-character-set=需要备份的数据库的字符集（备份出现乱码的时候用，一般不用）
    （5）备份单表或多表：前面不加-B参数 前面加数据库名，后接表名，就是备份单表或多表
    （6）-d 只备份表结构 -t 只备份数据 -T –tab=name 分离库表和数据成不同的文件，数据是文本，非SQL语句。
    （7）-F 刷新bin-log日志，重新生成新的bin-log文件。可以理解为切割bin-log
    （8）–master-data=1/2 在备份语句里添加change master语句和binlog文件对应的位置点。值为1为可执行的change master语句，为2是注释的change master语句。
    （9）–single-transaction 适合innodb事务数据库备份，保持数据一致性
    （10）-A 备份所有库
    （11）-x 锁表 –lock-all-tables -l –lock-tables 只读锁表
    （12）-R 备份额外的存储过程和函数

  • 生产场景，不同引擎mysqldump备份命令：
    myisam引擎企业生产备份命令（适合所有引擎或混合引擎）：
    mysqldump -uroot -p密码 -A -B -F -R–master-data=2 -x –events|gzip > /opt/bak$(date +%F).sql.gz (-F也可以不用，与–master-data有重复)
    innodb引擎备份命令：推荐使用：
    mysqldump -uroot -p密码 -A -B -F -R –master-data=2 –events –single-transaction|gzip > /opt/bak    $(date +%F).sql.gz (-F也可以不用，与–master-data有重复)

  • mysqldump命令逻辑备份，效率不是特别搞，简单、方便、可靠。适用与数据量不是特别大的场景，50G以内数据。 xtrabackup物理备份工具：全备和增量。 物理备份方案：从库停止SQL线程，打包，CP。

  • 数据库恢复命令 source 备份数据库文件路径.sql （执行前，确保字符集正确），需要先登录mysql，use库。
    数据库恢复标准命令 mysql -uroot -p密码 -S < 备份文件; （如果备份没有加-B参数，需要建库，use库，mysql后需加数据库名） gzip -d 解压文件，会删除压缩原文件。 gunzip <备份文件.sql.gz|mysql -u -p -S 此方法不会删除压缩源文件

  • 生产场景常用命令：
    show processlist; 查看正在执行的sql语句，看不全。
    show full processlist; 查看正在执行的完整sql语句，完整显示
    set global key_buffer_size = 32777218; 不重启数据库调整数据库参数，重启失效
    show variables; 查看数据库的参数信息，例如my.cnf参数生效情况
    kill ID 杀掉线程的命令，ID为线程号
    show status; 查看当前回话的数据库状态信息
    show global status; 查看整个数据库运行状态信息，很重要，要分析并做好监控。（工具mysqlreport）
    show engine innodb status\G 显示INNODB引擎的性能状态（早起版本show innodb status;）

  • mysqladmin的相关命令：
    mysqladmin -uroot -p123456 status
    mysqladmin -uroot -p123456 -S /data/3306/mysql.sock -i 1 status
    mysqladmin -uroot -p123456 flush-logs
    mysqladmin -uroot -p123456 processlist
    mysqladmin -uroot -p123456 extended-status
    mysqladmin -uroot -p123456 processlist -i 1 实时跟踪。
    watch mysqladmin -uroot -p123456 -S /data/3306/mysql.sock processlist

  • mysqlbinlog -d 参数解析指定库的binlog日志 -r 生成文件，相当于重定向”>”
    mysqlbinlog –base64-output=”decode-rows” –verbose 解析行模式下binlog日志
    mysqlbinlog mysql-bin.000020 –start-position=365 –stop-position=456 -r pos.sql
    mysqlbinlog mysql-bin.000020 –start-datetime=’2014-10-16 17:14:15’ –stop-datetime=’2014-10-16 17:15:15’ -r time.sql
